Objectively Awesome: Fork Bombs

Hey Ram, what the fork is a Fork Bomb?

Excellent question. Let me ask you another one. Are you familiar with Australia's Cane Toad Infestation? If not, here's a quick refresher. In the 1930's, Australia introduced a family of Cane Toads to the island in order to control the rampantly growing grey-backed Cane beetle population. The toads did NOT do this, and instead became their own menace to Australia. Australians still have to deal with Cane toads to this day.

You are probably wondering, "Ram, what does this have to do with Fork Bombs?" Well, kind of everything. To bit it simply, Fork Bombs are exactly like what happened in Australia, except they're happening in your machines with CPU processes rather than toads (I can't even imagine what it would be like to have TOADS in your computer).


To see an example of a fork bomb, look at this UNIX shell input:

:(){ :|:& };:

A quick explanation of what's happening here:

  • First, an explanation of each snippet of that command:
    • :() - This statement is defining some random function where our code will live
    • { - Begins the function, everything within this and its sister bracket will determine what happens
    • : - Loads a copy of the function that we are currently defining into memory
    • | - This signifies that the output of the function copy we loaded in previously is being piped to...
    • : - ...Another copy of the function that we're defining
    • & - This signifies that the statements that were declared to the immediate left of it should still be run in the background even after this command has been terminated
    • } - This ends the function that we just defined
    • ; - This ends our declaration of the function
    • : - Us calling the function we just defined. This begins the chaos that we have been orchestrating before this.
  • Whenever you type that command into your UNIX shell, you are starting a chain reaction of endless, worthless functions that exist only to generate two more like it.
  • This leads to these processes crowding your CPU with so many meaningless tasks that it will begin to hinder its ability to do actual work that you need it to do.
  • Only way to fix this is to reboot your computer. Not bad, but could be deadly if you need to save a ton of files. Long story short, try not to fall for this :).
